# PRD: YouTube Ad Detector MVP (rule-based)

## 목표
건강식품·다이어트 유튜브 광고성 영상의 **한국어 자막**에서 위법 소지가 있는 표현을 rule-based로 탐지해, 시청을 방해하지 않는 2-state 배지(`safe` / `fraud`)로 사용자에게 실시간 경고하고, `fraud` 판정 시 새 탭으로 보고서 페이지 링크를 제공한다. 본 시스템은 합법/불법을 최종 판정하는 기계가 아니라 **탐지기 + 근거 제시기 + 선별기**이며, 최종 판단은 사용자 몫이다.

## 사용자
건강식품·다이어트 관련 유튜브 영상을 시청하는 한국어 사용자. 건강기능식품·의료기기·의약품 과장 광고로 인한 소비자 피해를 줄이고자 하는 일반 시청자.

## 핵심 기능
1. **videoId 감지 및 한국어 자막 추출 (확장 단독)** — 유튜브 시청 페이지에서 videoId를 감지하고 `ytInitialPlayerResponse`로 한국어 자막 트랙을 로드한다. 자막이 없으면 UI를 표시하지 않고 스킵한다.
2. **rule-based 분석 및 3-zone 판정** — `@yad/shared`의 `analyze()`를 확장 내부에서 호출해 카테고리별 가중치를 합산하고 `safe` / `fraud` 판정을 낸다. 네트워크 왕복 없이 로컬에서만 수행한다.
3. **2-state 배지 렌더** — 시청 방해 없는 위치에 배지를 표시한다. `safe`는 최소 크기(또는 미표시), `fraud`는 경고 색상 + "자세히 보기" 링크를 노출한다.
4. **보고서 링크 (새 탭)** — `fraud` 판정 시 사용자가 링크를 누르면 background가 `chrome.tabs.create`로 `/report/{videoId}/{ruleVersion}` 경로를 새 탭에 연다. MVP에서는 메타 정보만 담은 스텁 페이지를 렌더한다.

## 비기능 요구사항
- **프라이버시**: 자막 원문·rule 매칭 결과를 외부 서버로 전송하지 않는다. 보고서 페이지는 `videoId`·`ruleVersion`만 URL로 받고, MVP에서는 서버가 매칭 결과를 저장하지 않는다.
- **성능**: videoId 감지부터 배지 렌더까지 일반 영상(자막 길이 ≤ 1만 자 기준) 500ms 이내를 목표로 한다. 유튜브 재생 성능에 간섭하지 않는다.
- **Fail-safe**: 자막 fetch 실패·파싱 실패·rule 엔진 예외는 모두 UI 미표시로 폴백한다. 어떤 경우에도 유튜브 플레이어·페이지를 방해하지 않는다.
- **보안 권한 최소화**: 호스트 권한은 `*.youtube.com`에 한정한다.

## 성공 지표 (MVP)
- 배포: Chrome Web Store 심사 통과 (MVP 1차 배포)
- 정확도(개발자 seed 샘플 20~30개 기준): 강한 위법 표현이 명백한 영상에서 `fraud` 판정률 ≥ 80%, 명백한 정상 영상에서 `fraud` 오탐률 ≤ 20%
- 파이프라인 무결성: 자막 있는 영상에서 배지 렌더 성공률 ≥ 95% (내부 테스트)
- 위 지표는 포스트-MVP에서 실사용자 피드백·로그 기반 재튜닝의 출발점일 뿐, 절대 기준이 아니다.

## 범위 외 (Out of Scope)
- 제품 진위 판별·의학적 사실 검증
- 합법/불법 최종 판정 (본 시스템은 선별·근거 제시에 한정)
- 건강식품·다이어트 외 카테고리 탐지
- AI 모델 2차 판별 (LLM vs 파인튜닝 미결정, 포스트-MVP)
- LLM 기반 보고서 본문 생성
- 보고서 페이지 실제 렌더(findings·자막·법조항 나열)
- 보고서 업로드·저장·공유 영속화
- 자막 없는 영상 분석
- 한국어 외 언어 자막
- 서버 측 자막 수집·rule 엔진 실행
- 광고 차단·삭제·광고 차단 우회 대응
- 사용자 피드백 수집·텔레메트리
